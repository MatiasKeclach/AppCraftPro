1️⃣ Stack & Tecnologías

  • Backend: Node.js + Express.js + Socket.io
  • Base de Datos: SQLite (local) / PostgreSQL (producción) con Supabase
  • Frontend: React + Next.js + Tailwind CSS
  • CMS: Propio con roles, gestión de stock, productos y pedidos
  • Automatización: JSON para carga masiva de productos y fotos
  • Hosting: Render + Supabase
  • Integraciones: Email, notificaciones críticas y IA para predicción de stock

2️⃣ Roles del Sistema

  • Owner / Founder: Control total de todas las tiendas y supervisión de superadmins.
  • Superadmin: Cliente principal de la tienda, ciertas acciones críticas requieren autorización del Owner.
  • Admin: Gestión de productos, stock, pedidos y clientes.
  • Cliente: Usuario final, compra productos y consulta stock.

3️⃣ CRUD & Gestión de Productos

  • Crear, leer, actualizar y eliminar productos dinámicamente.
  • Stock por producto, talle y color, con código único (SKU).
  • Alertas automáticas de “sin stock”.
  • Carga masiva vía JSON.
  • Galerías de imágenes vinculadas automáticamente.
  • Soporte para curvas de talles y venta mayorista/minorista.

4️⃣ Automatización con JSON

Ejemplo de JSON para cargar productos automáticamente:

{
  "productos": [
    {
      "nombre": "Remera Básica",
      "categoria": "Remeras",
      "talle": "M",
      "color": "Rojo",
      "precio": 1500,
      "stock": 20,
      "imagenes": ["remera-roja-1.jpg","remera-roja-2.jpg"]
    },
    {
      "nombre": "Jeans Slim",
      "categoria": "Pantalones",
      "talle": "32",
      "color": "Azul",
      "precio": 3500,
      "stock": 15,
      "imagenes": ["jeans-azul-1.jpg","jeans-azul-2.jpg"]
    }
  ]
}

El sistema interpreta este JSON, genera SKUs automáticos y actualiza el stock.

6️⃣ Explicación para el Cliente

Podés gestionar productos, ver stock y recibir alertas automáticas cuando algo se agota.

Podés cargar productos en masa usando JSON y el sistema genera todo automáticamente.

Distintos roles permiten que cada usuario tenga permisos adecuados y yo, como Owner, superviso todo.

Esto permite escalar la plataforma y replicar nuevas tiendas sin rehacer código.
