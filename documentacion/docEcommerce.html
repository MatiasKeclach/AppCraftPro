Documentación Técnica - Tienda Virtual

Proyecto CMS y tienda virtual escalable con automatización y roles diferenciados

Stack & Tecnologías Roles CRUD & Productos Automatización JSON Arquitectura ASCII Explicación Cliente Otro

1️⃣ Stack & Tecnologías

  • Backend: Node.js + Express.js + Socket.io
  • Base de Datos: SQLite para local / PostgreSQL en producción
  • Frontend: React + Next.js + Tailwind CSS
  • CMS: Sistema propio integrado con roles y gestión de stock
  • Automatización: JSON para carga masiva de productos y fotos
  • Hosting: Render + Supabase para DB en nube
  • Integraciones: Email notificaciones, IA para predicción de stock y ventas

2️⃣ Roles del Sistema

  • Owner / Founder: Gestión completa de todas las tiendas, supervisión de superadmins, notificaciones críticas.
  • Superadmin: Cliente principal de la tienda, puede gestionar todo el CMS pero ciertas acciones críticas requieren autorización del Owner.
  • Admin: Personal autorizado para gestión de productos, stock, pedidos y clientes.
  • Cliente: Usuario final que compra productos, visualiza stock y realiza pedidos.

3️⃣ CRUD & Gestión de Productos

  • Crear, leer, actualizar y eliminar productos de forma dinámica.
  • Stock por producto, talle y color, con código único de identificación (SKU).
  • Alertas automáticas de sin stock cuando no quedan unidades.
  • Posibilidad de carga masiva vía JSON.
  • Galerías de imágenes vinculadas automáticamente a productos.
  • Permite curvas de talles y venta mayorista / minorista.

4️⃣ Automatización con JSON

Ejemplo de JSON para cargar productos automáticamente:

{
  "productos": [
    {
      "nombre": "Remera Básica",
      "categoria": "Remeras",
      "talle": "M",
      "color": "Rojo",
      "precio": 1500,
      "stock": 20,
      "imagenes": ["remera-roja-1.jpg", "remera-roja-2.jpg"]
    },
    {
      "nombre": "Jeans Slim",
      "categoria": "Pantalones",
      "talle": "32",
      "color": "Azul",
      "precio": 3500,
      "stock": 15,
      "imagenes": ["jeans-azul-1.jpg", "jeans-azul-2.jpg"]
    }
  ]
}

Al importar este JSON, el sistema asigna automáticamente códigos SKU y actualiza el stock de cada producto.

6️⃣ Explicación para el Cliente

Esta tienda virtual está pensada para que puedas gestionar tus productos de manera muy simple. Cada producto tiene un código único y se puede ver la cantidad de stock disponible. Si se agota un producto, el sistema automáticamente muestra “sin stock”.

Podés cargar productos de forma masiva usando un archivo JSON que contiene toda la información de cada producto: nombre, categoría, talle, color, precio y fotos. El sistema lo interpreta y genera todo automáticamente.

Hay distintos roles: como cliente podés comprar, como admin podés gestionar productos y pedidos, como superadmin podés gestionar toda la tienda, y yo, como Owner, superviso todo el sistema para evitar errores graves y mantener la plataforma funcionando sin problemas.

Esto permite que el sistema sea escalable, seguro y que funcione de forma automática, ahorrando tiempo y evitando errores. Además, si algún día querés otra tienda virtual, podemos replicar toda esta arquitectura sin necesidad de rehacer nada desde cero.
