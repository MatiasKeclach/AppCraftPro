💡 Introducción

Esta documentación explica la creación y funcionamiento de la Fábrica Digital de Tiendas Virtuales. Permite instalar tiendas virtuales completas usando un único comando por consola, con CMS y automatización integrada.

🏗️ Arquitectura

El kernel del sistema funciona como motor principal. Integra frontend, backend, base de datos y CMS propio, permitiendo una instalación automatizada y completa de la tienda virtual.

🤖 Inteligencia Artificial

1️⃣ Qué es la Inteligencia Artificial (IA)

La Inteligencia Artificial (IA) es un sistema que imita procesos de inteligencia humana para tomar decisiones y automatizar tareas. En nuestra fábrica de tiendas e-commerce, la IA analiza datos de la tienda y optimiza la configuración, las sugerencias y los procesos automáticamente.

2️⃣ Cómo funciona la IA en la tienda e-commerce

  • Alta automática de productos: sugiere categorías, precios y etiquetas optimizadas.
  • Optimización de inventario: predice qué productos venderán más y cuándo reponer stock.
  • Recomendaciones personalizadas: analiza el comportamiento del usuario y genera sugerencias automáticas.
  • Automatización de marketing: campañas de emails o notificaciones automáticas según patrones de compra.
  • Análisis de rendimiento: reportes sobre ventas, productos más populares y comportamiento del usuario.

3️⃣ Integración práctica en la fábrica

Cada tienda creada desde la fábrica tiene habilitada la IA por defecto. Funciona en segundo plano mediante el módulo de predicción, que interactúa con la base de datos y la API. Los desarrolladores pueden activar o desactivar la IA por tienda, o personalizar su comportamiento mediante la configuración avanzada.

4️⃣ Ventajas

  • Ahorro de tiempo: la tienda se configura automáticamente de manera inteligente.
  • Mejora en ventas: recomendaciones y optimización de stock.
  • Personalización: cada usuario recibe una experiencia adaptada.
  • Escalabilidad: se puede replicar en múltiples tiendas sin intervención manual.

5️⃣ Desventajas / Consideraciones

  • Requiere datos correctos: productos o usuarios incompletos generan predicciones incorrectas.
  • No reemplaza al humano: la IA es asistente, no sustituto total.
  • Consumo extra de recursos: habilitar en exceso múltiples tiendas simultáneamente puede ralentizar el sistema.

6️⃣ Errores comunes

  • Desactivar la IA sin revisar la configuración avanzada → pérdida de predicciones.
  • Subir productos sin datos completos → sugerencias incorrectas.
  • Ignorar actualizaciones del módulo de predicción → resultados menos precisos.

7️⃣ Buenas prácticas

  • Revisar y completar los datos de productos y usuarios para mejorar las predicciones.
  • Monitorear el rendimiento de la IA y ajustar la configuración según sea necesario.
  • Combinar la IA con la supervisión humana para obtener los mejores resultados.

🛡 ⚙️ Roles y Permisos

En la fábrica, los roles definen los niveles de acceso y operaciones permitidas:

  • Owner: Acceso total. Puede generar API Key, crear tiendas y administrar roles.
  • Superadmin: Gestiona tiendas, usuarios y contenidos, pero no puede generar la API Key del Owner.
  • Admin: Acceso limitado a gestión de contenido y operaciones dentro de tiendas asignadas.
  • Cliente: Usuarios que consumen la tienda.

Ejemplo de verificación de rol antes de ejecutar una acción:

// Verificación de rol antes de ejecutar acción if (usuario.rol === 'Owner') { ejecutarAccion(); } else if (usuario.rol === 'Superadmin') { ejecutarAccionLimitada(); } else { denegarAccion(); }

Seguridad

La Fábrica de Tiendas prioriza la seguridad de datos, accesos y transacciones.

  • API Key privada: Cada owner genera su API Key única. Nunca compartirla.
  • Validación por IP: Solo solicitudes desde servidores autorizados son aceptadas.
  • Control de errores: Validación de roles y permisos antes de realizar acciones críticas.
  • Cifrado de datos: Toda la información sensible se almacena y transmite cifrada.

Errores comunes a evitar:

  • Exponer la API Key públicamente.
  • No limitar solicitudes a IPs confiables.
  • No validar roles antes de modificar datos críticos.

🏭 Fábrica de Tiendas y Herramienta de Automatización

Esta sección describe la creación de la herramienta que automatiza la generación de tiendas virtuales, permitiendo clonar de forma rápida y consistente cualquier proyecto.

Concepto

La fábrica de tiendas es una herramienta que, a partir de un único comando en consola, genera la estructura completa de una tienda virtual. Esto incluye:

  • Instalación del stack completo (backend, base de datos, front-end, CMS).
  • Generación de roles y permisos predefinidos.
  • Integración de la API para automatización y administración.
  • Front-end base listo para personalizar y agregar productos, categorías y filtros.

Flujo recomendado de creación

Para crear la fábrica de tiendas de manera eficiente, se recomienda seguir este flujo:

  1. Instalar todo el stack: configurar servidor, base de datos, PHP/Node.js (según tu elección), front-end base y CMS.
  2. Crear la tienda base: generar la estructura inicial de la tienda dentro del stack, con CMS y roles predefinidos.
  3. Verificar que todo funcione: probar front-end, back-end, permisos de usuario, API y CMS.
  4. Crear el comando de la fábrica: escribir un script que, al ejecutarse, genere automáticamente una nueva tienda con todas las configuraciones predeterminadas.
  5. Pruebas del comando: ejecutar la herramienta para asegurarse de que la tienda se crea correctamente, incluyendo CMS, roles, API y front-end base.

Soporte para múltiples tecnologías

Si en el futuro quieres crear tiendas con otra tecnología (por ejemplo, distinto framework de front-end, base de datos, o CMS), simplemente se debe generar otro comando específico para esa tecnología. Esto permite mantener la fábrica flexible y adaptable.

Recomendaciones importantes

  • Siempre crear primero la tienda base y probarla antes de crear el comando.
  • Mantener un estándar de roles, API y estructura de carpetas para que el comando funcione de manera confiable.
  • Documentar cada comando creado para futuras referencias y posibles expansiones.
  • Si agregas funciones adicionales (carrito, filtros, animaciones, paginación), incluirlas en la tienda base antes de automatizar el comando.

Ejemplo de comando

# Crear nueva tienda con la fábrica fabrica_crear_tienda --nombre "TiendaDemo" --usuario "owner" --template "moderno" # Crear nueva tienda con otro stack/tecnología fabrica_crear_tienda_v2 --nombre "TiendaReact" --usuario "owner" --template "react"

De esta manera, cada vez que necesites generar una tienda, solo ejecutarás el comando correspondiente y la herramienta creará toda la estructura automáticamente.

El CLI (Command Line Interface) es el núcleo ejecutor de la fábrica de tiendas. Permite generar nuevas tiendas mediante comandos simples, automatizando toda la instalación del sistema.

📌 Concepto

El CLI actúa como un intérprete de órdenes que traduce comandos en acciones reales sobre el sistema: creación de carpetas, instalación de dependencias, configuración del entorno y despliegue inicial.

Usuario → Comando → CLI → Motor → Tienda creada

--force            → Sobrescribe si ya existe

⚙️ Comando create-store


const fs = require('fs-extra');
const path = require('path');

module.exports = async function(args) {

  const storeName = args[1];

  if (!storeName) {
    console.log('⚠️ Debes especificar un nombre para la tienda');
    return;
  }

  const targetPath = path.join(process.cwd(), storeName);

  if (fs.existsSync(targetPath)) {
    console.log('❌ La carpeta ya existe');
    return;
  }

  try {
    console.log('🚀 Creando tienda...');

    await fs.copy(
      path.join(__dirname, '../templates/base'),
      targetPath
    );

    console.log('✅ Estructura creada');

    // Crear archivo .env básico
    const envContent = `
PORT=3000
DB_NAME=${storeName}
DB_USER=root
DB_PASS=
    `;

    fs.writeFileSync(path.join(targetPath, '.env'), envContent);

    console.log('✅ Configuración generada');

    console.log('🎉 Tienda creada con éxito');

  } catch (error) {
    console.log('❌ Error al crear la tienda');
    console.error(error);
  }

};

🧠 Comando interactivo (createStore.js)


const fs = require('fs-extra');
const path = require('path');
const inquirer = require('inquirer');
const chalk = require('chalk');
const ora = require('ora');

module.exports = async function() {

  console.log(chalk.cyan('\n🧠 MattDev CLI - Generador de Tiendas\n'));

  const answers = await inquirer.prompt([
    {
      type: 'input',
      name: 'storeName',
      message: 'Nombre de la tienda:',
      validate: input => input ? true : 'Debes ingresar un nombre'
    },
    {
      type: 'list',
      name: 'stack',
      message: 'Selecciona el stack:',
      choices: ['node', 'php']
    },
    {
      type: 'list',
      name: 'template',
      message: 'Selecciona template:',
      choices: ['basic', 'pro']
    },
    {
      type: 'confirm',
      name: 'installDeps',
      message: '¿Instalar dependencias automáticamente?',
      default: true
    }
  ]);

  const targetPath = path.join(process.cwd(), answers.storeName);

  if (fs.existsSync(targetPath)) {
    console.log(chalk.red('❌ La carpeta ya existe'));
    return;
  }

  const spinner = ora('Creando tienda...').start();

  try {
    await fs.copy(
      path.join(__dirname, ../templates/${answers.template}),
      targetPath
    );

    spinner.text = 'Generando configuración...';

    const envContent = `
PORT=3000
STACK=${answers.stack}
DB_NAME=${answers.storeName}
DB_USER=root
DB_PASS=
    `;

    fs.writeFileSync(path.join(targetPath, '.env'), envContent);

    spinner.succeed('✅ Tienda creada correctamente');

    if (answers.installDeps) {
      console.log(chalk.yellow('\n📦 Instalando dependencias...'));

      const { execSync } = require('child_process');
      execSync('npm install', { cwd: targetPath, stdio: 'inherit' });

      console.log(chalk.green('✅ Dependencias instaladas'));
    }

    console.log(chalk.green('\n🎉 Proyecto listo para usar\n'));

  } catch (error) {
    spinner.fail('❌ Error al crear la tienda');
    console.error(error);
  }

};

module.exports = async function(context) {

  const { targetPath } = context;

  console.log('💳 Instalando sistema de pagos...');

  const fs = require('fs-extra');
  const path = require('path');

  const paymentsPath = path.join(targetPath, 'payments');

  await fs.ensureDir(paymentsPath);

  fs.writeFileSync(
    path.join(paymentsPath, 'payment.js'),
    '// lógica de pagos'
  );

  console.log('✅ Plugin de pagos instalado');

};

🧩 Módulo Futuro: Generador de Documentos (WYSIWYG)

Se plantea la implementación de un módulo dentro de la plataforma (AppCraft) que permita la creación de documentos estructurados mediante un editor visual tipo WYSIWYG (What You See Is What You Get).

El objetivo es permitir que el usuario genere contenido sin necesidad de conocimientos técnicos, escribiendo de forma similar a herramientas como Word, pero obteniendo como resultado documentos con formato profesional listos para web o exportación.

🎯 Objetivos

  • Permitir la creación de documentos sin escribir código
  • Separar estructura (template) de contenido (input del usuario)
  • Generar documentos automáticamente a partir de plantillas predefinidas
  • Facilitar exportación y publicación

⚙️ Funcionalidades principales

  • Editor WYSIWYG embebido
  • Sistema de secciones dinámicas
  • Inserción de contenido estructurado (títulos, bloques, listas, código)
  • Render automático en formato visual tipo “manual profesional”
  • Vista previa en tiempo real
