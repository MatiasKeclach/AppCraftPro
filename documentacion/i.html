Tienda Virtual y CMS - Manual de Proyecto

Presentación del sistema, roles, arquitectura y funcionalidades.

1️⃣ Roles y Permisos

El sistema maneja distintos niveles de acceso para asegurar seguridad, control y escalabilidad:

  • Owner: Control total del sistema, recibe alertas críticas, puede aprobar o revertir cambios de superadmins.
  • Superadmin: Cada tienda tiene su superadmin, puede gestionar productos, stock y usuarios, pero limitado en cambios críticos.
  • Admin: Gestiona partes del CMS según permisos asignados por superadmin.
  • Cliente: Usuario final que puede comprar en la tienda virtual y consultar su historial de pedidos.

2️⃣ Lógica de Backend y Funcionalidades

El CMS y la tienda virtual funcionan de forma integrada:

  • CRUD completo de productos: agregar, editar, eliminar productos.
  • Gestión de stock: cada producto tiene color, talle, código único y cantidad disponible.
  • Notificación automática si un superadmin intenta cambios críticos.
  • Carga masiva de productos mediante JSON: fotos, categorías, variantes y precios se insertan automáticamente.
  • Automatización de estado de stock: si se agota, el producto se marca automáticamente como “Sin stock”.
  • Conexión en tiempo real entre CMS y tienda mediante WebSockets para reflejar cambios instantáneamente.

3️⃣ Arquitectura del Sistema

El sistema está estructurado para ser escalable, seguro y eficiente:

  • Frontend: Next.js + React + Tailwind CSS, para UI rápida, responsive y moderna.
  • Backend: Node.js + Express para manejar rutas, APIs y lógica de negocios.
  • Base de datos: SQLite para desarrollo local, migrable a PostgreSQL en producción con Supabase.
  • Automatización e IA: IA para analizar stock, generar códigos únicos, alertas y recomendaciones.
  • Integración JSON: Facilita la carga masiva de productos, categorías y variantes.
  • CMS y tienda virtual: Conectados en tiempo real, reflejando cambios instantáneamente.

4️⃣ Tecnologías Usadas

Node.js Express Next.js React Tailwind CSS SQLite PostgreSQL SupaBase WebSockets JSON IA / Automatización

5️⃣ Explicación para el Cliente

En palabras simples, esto es lo que hace tu tienda y CMS:

  • Cada tienda tiene su panel de control (CMS) donde el superadmin puede agregar productos, talles, colores y fotos.
  • El sistema crea un código único por producto, lo que permite gestionar el stock automáticamente.
  • Si algún producto se agota, se muestra como “Sin stock” en la tienda automáticamente.
  • Si el superadmin intenta hacer un cambio crítico, el Owner recibe una notificación y decide si se autoriza.
  • Todo el proceso se puede automatizar para que, por ejemplo, con un archivo JSON se carguen decenas o cientos de productos de una sola vez.
  • La tienda virtual está conectada en tiempo real con el CMS, así los cambios se reflejan al instante.
  • El sistema está pensado para crecer: si mañana hay 100 o 200 tiendas, cada una puede gestionarse de forma independiente, pero vos siempre tendrás control total.
  • Además, la tecnología usada permite que la tienda sea rápida, segura y moderna, y que se pueda mantener y escalar sin rehacer nada.
