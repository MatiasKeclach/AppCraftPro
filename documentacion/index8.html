🚀 Infraestructura Profesional

Guía completa para pasar de desarrollador a creador de sistemas escalables y con ingresos recurrentes.

🖥️ VPS (Servidor)

Un VPS es tu propio servidor en internet. Acá vas a alojar todos tus sistemas.

  • Contratar VPS (DigitalOcean / Vultr)
  • Acceso root por SSH
  • Instalar Node.js para correr tu backend
  • Instalar NGINX como servidor web

Objetivo: tener control total sobre tu infraestructura y no depender de terceros.

🌐 Dominios

Cada cliente necesita una dirección web. Esto se conecta con tu servidor.

  • Comprar dominio (Namecheap / GoDaddy)
  • Configurar DNS → apuntar a IP del VPS
  • Usar subdominios para múltiples clientes
cliente1.tusistema.com cliente2.tusistema.com

Estrategia: sistema multi-tenant para escalar sin duplicar código.

🗄️ Base de Datos

Donde se guarda toda la información de los clientes.

  • Inicio: SQLite (simple y rápido)
  • Escalado: PostgreSQL o MySQL
  • Separación lógica con tenant_id

Clave: empezar simple y migrar cuando crezcas.

🔐 Seguridad

Esto es lo que te diferencia de un desarrollador amateur.

  • HTTPS con certificados SSL
  • Hash de contraseñas (bcrypt)
  • Validación de datos en backend
  • Protección contra SQL Injection y XSS

Objetivo: que tu sistema no se rompa ni sea vulnerable.

💾 Backups

Si perdés datos, perdés clientes. Esto es crítico.

  • Backup automático diario
  • Guardar copias fuera del servidor
  • Restauración rápida

Nivel pro: backups versionados + recuperación en minutos.

⚙️ Deploy

Proceso para subir tu sistema a producción.

  • Subir código con Git
  • Instalar dependencias
  • Usar PM2 para mantener servidor activo
  • NGINX como proxy inverso
Cliente → Dominio → NGINX → Node API → Base de Datos

Resultado: sistema online, accesible y estable.

🛠️ Implementación Técnica Paso a Paso

Guía real para levantar tu infraestructura desde cero.

1. Crear VPS

  • Elegir proveedor (DigitalOcean / Vultr)
  • Seleccionar Ubuntu 22.04
  • Copiar IP del servidor
ssh root@IP_DEL_SERVIDOR

2. Configuración inicial

apt update && apt upgrade -y apt install nginx git curl -y

3. Instalar Node.js

curl -fsSL https://deb.nodesource.com/setup_18.x | bash - apt install -y nodejs node -v

4. Clonar proyecto

git clone https://tu-repo.git cd tu-repo npm install

5. Ejecutar con PM2

npm install -g pm2 pm2 start app.js pm2 save pm2 startup

6. Configurar NGINX

server { listen 80; server_name tudominio.com;location / { proxy_pass http://localhost:3000; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; } }

7. Activar dominio

  • Ir a panel de dominio
  • Apuntar A record → IP VPS

8. SSL (HTTPS)

apt install certbot python3-certbot-nginx -y certbot --nginx -d tudominio.com

9. Base de datos

SQLite:

const db = new Database('database.db');

Escalado: migrar a PostgreSQL

10. Backups automáticos

crontab -e0 2 * * * cp database.db backup-$(date +%F).db

11. Seguridad básica

  • Usar dotenv
  • Hash passwords (bcrypt)
  • Validar inputs
  • Helmet.js

12. Multi-cliente (tu máquina)

Arquitectura base:

tenant_id → separa clientes subdominio → identifica cliente config dinámica → carga tienda

👥 Multi-Tenant (núcleo del SaaS)

Cada cliente usa el mismo sistema pero con sus propios datos.

tenant_id
  • Cada request identifica cliente
  • Se filtran datos automáticamente

🗄️ Estrategias de Base de Datos

  • Simple: una DB con tenant_id
  • Intermedio: schema por cliente
  • Pro: DB separada por cliente

Recomendado: empezar simple y escalar.

🌐 Identificación por dominio

NGINX o backend detecta el host y carga config.

📁 Estructura CLI

/create-matt-app ├── bin/ │ └── index.js ├── templates/ │ ├── tienda/ │ ├── academia/ ├── package.json

⚙️ Archivo ejecutable

#!/usr/bin/env nodeconst fs = require('fs'); const path = require('path'); const projectName = process.argv[2]; if(!projectName){ console.log('❌ Poné un nombre'); process.exit(1); } const target = path.join(process.cwd(), projectName); fs.mkdirSync(target); console.log('🚀 Proyecto creado:', projectName);

📦 package.json CLI

{ "name": "create-matt-app", "bin": { "create-matt-app": "bin/index.js" } }

🔗 Linkear CLI local

npm link

Ahora podés usar:

create-matt-app mi-proyecto
